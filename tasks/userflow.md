## 拖曳檔案到 Pod 觸發 Agent

### 情境：使用者拖曳檔案到 idle 的串行 Pod
- Given 使用者開啟 Canvas 且某個串行 Pod 的 agent 處於 idle 狀態
- When 使用者從 Finder 拖曳一個或多個有效檔案放到該 Pod 上
- Then 檔案被收下並自動觸發該 Pod 的對話，agent 開始處理這些檔案

### 情境：使用者拖曳檔案到忙碌中的串行 Pod
- Given 使用者開啟 Canvas 且某個串行 Pod 的 agent 正在忙碌中
- When 使用者把檔案拖到該 Pod 上方
- Then Pod 不接受拖曳，使用者無法把檔案丟進去

### 情境：使用者拖曳檔案到 Multi-Instance Pod
- Given 使用者開啟 Canvas 且目標 Pod 是 Multi-Instance Pod
- When 使用者拖曳檔案到該 Pod 上（不論是否已有其他 Run 在跑）
- Then 檔案被收下並建立一個新的 Run，agent 開始處理這些檔案

### 情境：使用者拖曳了 0 個檔案或拖曳資料夾
- Given 使用者準備拖東西到 Pod 上
- When 使用者拖曳的內容是空的，或是包含資料夾
- Then 系統不接受這次拖曳，並顯示提示訊息告知使用者無法處理

### 情境：使用者拖曳的檔案總大小超過 20 MB
- Given 使用者準備把檔案拖到 Pod 上
- When 使用者拖曳的檔案整批大小超過 20 MB
- Then 系統拒絕這次拖曳並顯示提示訊息告知使用者超過大小限制

### 情境：寫入檔案時磁碟空間不足
- Given 使用者拖曳了有效大小的檔案到 Pod 上
- When 系統嘗試把檔案寫入暫存區，但磁碟空間不足
- Then 系統拒絕這次拖曳並顯示提示訊息告知使用者磁碟空間不足，對話不會被建立

### 情境：使用者拖曳了多個同名檔案
- Given 使用者從不同來源資料夾選了名稱相同的檔案
- When 使用者把這些同名檔案一起拖到 Pod 上
- Then 系統自動為重複的檔名加上後綴避免衝突，全部檔案都成功落地

### 情境：寫入檔案過程中發生錯誤
- Given 使用者拖曳了檔案到 Pod 上
- When 系統在寫入檔案的過程中發生錯誤
- Then 系統清掉已經寫入的暫存內容、不建立任何對話訊息，並顯示提示訊息告知使用者操作失敗

### 情境：拖曳成功並自動觸發對話
- Given 使用者成功把檔案拖到 Pod 上
- When 系統完成檔案落地
- Then 系統以使用者身分送出一則訊息，告知 agent 提供了哪些檔案以及檔案的絕對路徑，agent 接著開始處理

### 情境：暫存檔案經過 24 小時後自動清除
- Given 使用者過去拖曳到 Pod 的檔案被存放在系統暫存區
- When 暫存檔案的最後修改時間超過 24 小時
- Then 系統自動清除這些過期的暫存檔案

### 情境：使用者拖曳檔案進出 Pod 範圍時的視覺回饋
- Given 使用者正在從作業系統拖曳檔案
- When 使用者把檔案拖入 Pod 的範圍
- Then 該 Pod 顯示高亮狀態，提示使用者可以放開放下檔案

### 情境：使用者拖曳檔案離開 Pod 範圍
- Given 使用者拖曳的檔案目前正懸停在 Pod 上方且 Pod 已高亮
- When 使用者把檔案移出 Pod 範圍
- Then 該 Pod 取消高亮回到一般狀態

### 情境：使用者一次拖曳多種類型的檔案
- Given 使用者準備同時提供圖片、文字檔、二進制檔給 agent
- When 使用者把這些不同類型的檔案一起拖到 Pod 上
- Then 系統全部接受並落地，agent 收到觸發訊息後自行讀取與處理這些檔案
